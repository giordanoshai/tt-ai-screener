--- SYSTEM ARCHITECTURE ---

The system has 3 analysis skills. Each skill receives a JSON data payload automatically — the prompt only needs to define the analysis logic and output format. The user DOES NOT need to write SQL or fetch data; the system handles all data assembly.

Skill 1: Long-Term Screen (longterm_screen)

Purpose: Screen for quality stocks worth holding 6-12 months. Fundamentals-driven.
Input JSON structure:

{
  "date": "2026-06-17",
  "source": "built-in criteria (...)" | "screener (N tickers)",
  "candidates_count": 50,
  "candidates": [
    {
      "ticker": "AAPL", "company_name": "Apple Inc.", "sector": "Technology",
      // --- Fundamentals ---
      "revenue_growth_yoy": 0.25,     // YoY revenue growth (0.25 = 25%)
      "earnings_growth_yoy": 0.30,    // YoY earnings growth
      "gross_margin": 0.45,           // Gross margin ratio
      "pe_ratio": 28.5,               // Price-to-Earnings
      "ps_ratio": 7.2,                // Price-to-Sales
      "pb_ratio": 12.1,               // Price-to-Book
      "peg_ratio": 1.1,               // PEG (PE / growth), <1 = undervalued
      "market_cap": 3200000000000,    // Market cap in USD
      "roe": 0.35,                    // Return on Equity
      "fcf_yield": 0.04,             // Free Cash Flow Yield
      // --- Analyst ---
      "analyst_rating": "buy",        // buy | hold | sell | strong_buy | strong_sell
      "analyst_target_price": 210.0,  // Consensus target price
      "analyst_count": 32,            // Number of analysts covering
      // --- Insider ---
      "mspr": 15.3,                   // Month Share Purchase Ratio (Finnhub)
                                      // >0 = net buying, <0 = net selling, null = no data
      // --- Earnings ---
      "next_earnings_date": "2026-07-25",
      "last_earnings_date": "2026-04-30",
      // --- Technical (latest trading day) ---
      "close": 195.5,                 // Latest close price
      "ma_20": 190.0,                 // 20-day moving average
      "ma_50": 185.0,                 // 50-day MA
      "ma_200": 170.0,               // 200-day MA
      "rsi_14": 55.2,                // RSI(14), 30=oversold, 70=overbought
      "dist_ma20_pct": 0.029,        // Distance from MA20 (0.029 = +2.9%)
      "dist_ma50_pct": 0.057,        // Distance from MA50
      "vol_ratio": 1.35,             // Volume / 20-day avg volume
      "atr_pct": 0.023,              // ATR(14) / close, daily volatility
      "high_20": 198.0,              // 20-day high
      "pct_chg": 0.012,              // Today's price change %
      // --- News (last 30 days, up to 5 per ticker) ---
      "recent_news": [
        {
          "headline": "Apple beats Q2 estimates...",
          "sentiment_label": "positive",  // positive | negative | neutral
          "sentiment_score": 0.72,        // AI score: -1.0 (very negative) to +1.0 (very positive)
          "published_at": "2026-06-15 14:30:00"
        }
      ]
    }
  ]
}

Skill 2: Swing Screen (swing_screen)

Purpose: Find 2-20 day swing trade setups. Technically driven.
Input JSON structure:

{
  "date": "2026-06-17",
  "source": "built-in criteria (...)" | "screener (N tickers)",
  "candidates_count": 100,
  "candidates": [
    {
      "ticker": "NVDA", "company_name": "NVIDIA Corp", "sector": "Technology",
      // --- Technical (primary, always present) ---
      "close": 120.5, "ma_20": 118.0, "ma_50": 110.0, "ma_200": 95.0,
      "rsi_14": 58.0,                // RSI(14), 30=oversold, 70=overbought
      "vol_ratio": 1.8,             // volume / 20-day avg, >1.5 = surge
      "atr_pct": 0.03,              // ATR(14)/close, daily volatility
      "dist_ma20_pct": 0.021,       // (close-MA20)/MA20, +=above MA
      "dist_ma50_pct": 0.095,
      "high_20": 122.0,             // 20-day high; close ≥ high_20*0.99 = breakout
      "pct_chg": 0.015,             // today's price change %
      // --- Fundamental risk check (may be null) ---
      "next_earnings_date": "2026-07-10",  // within 7 days = earnings risk
      "pe_ratio": 45.0,
      "revenue_growth_yoy": 0.22,
      // --- News (last 7 days, up to 3 per ticker) ---
      "recent_news": [
        { "headline": "...", "sentiment_label": "positive",
          "sentiment_score": 0.6, "published_at": "2026-06-15 14:30:00" }
      ]
    }
  ]
}

Source note: When built-in, candidates are pre-filtered (close>MA20 & >MA50, MA20>MA50, vol_ratio>1.2, RSI 45-75) and carry ONLY the fields above. When screener, candidates also include the full Long-Term fundamental block (analyst_*, mspr, gross_margin, peg_ratio, etc.) but are NOT trend-pre-filtered — the prompt should re-check trend itself.

Skill 3: Position Review (position_review)

Purpose: Evaluate existing holdings — continue, add, reduce, or exit.
Input JSON structure:

{
  "date": "2026-06-17",
  "positions_count": 3,
  "positions": [
    {
      "ticker": "AAPL", "company_name": "Apple Inc.",
      "sector": "Technology", "industry": "Consumer Electronics",
      // All technical + fundamental fields (same as above)
      // Plus:
      "cost_basis": 150.0,    // User's average entry price
      "shares": 20,           // Number of shares held
      // news: last 30 days, up to 20 per ticker (more for holdings)
      "news": [ { "headline": "...", "sentiment_label": "...", "sentiment_score": 0.5, "published_at": "..." } ]
    }
  ]
}

--- KEY DATA NOTES ---

  • mspr (Month Share Purchase Ratio): Finnhub insider metric. Positive = insiders buying, negative = selling. Often NULL for small/mid-caps — treat null as neutral, not negative.
  • sentiment_score: AI-scored from -1.0 to +1.0. Not all news has scores. Label can be positive/negative/neutral.
  • vol_ratio: Today's volume divided by 20-day avg. >1.5 = significant volume surge. <0.5 = quiet day.
  • atr_pct: ATR(14) / close price. Represents typical daily % move. Useful for stop-loss and target calculation.
  • dist_ma20_pct / dist_ma50_pct: (close - MA) / MA. Positive = above MA, negative = below. Useful for mean-reversion and trend strength.
  • high_20: 20-day high. close ≥ high_20 * 0.99 indicates breakout.
  • analyst_rating: String: buy, hold, sell, strong_buy, strong_sell. Can be null.
  • peg_ratio: PE / expected growth. <1 = undervalued relative to growth. Can be null or negative (meaningless if earnings negative).
  • Null values: Many fields can be null (especially for IPOs, small-caps, or tickers with limited coverage). Prompt should handle nulls gracefully.

--- PROMPT DESIGN RULES ---

  • The prompt is a system instruction given to an AI model. Data arrives in the user message as JSON.
  • DO NOT include SQL queries in the prompt — the system handles data fetching. The prompt only defines analysis logic.
  • Define clear scoring/rating criteria so the AI produces consistent, structured output.
  • Define an output format: per-ticker analysis blocks + a summary table. Use plain text (markdown-compatible), not HTML.
  • Keep the prompt under 6000 characters to leave room for data and AI response.
  • The AI has the role "professional stock analyst" — your prompt defines the methodology, not the persona.
  • Output language should match the prompt language (the system handles this automatically).
